# StateLessWidget
-----------------
1) 값이 바뀌는가? N
2) 누가 상태를 관리하는가? 상태 없음. 외부에서 값만 받음

: state(상태)가 없는 위젯 -> 한 번 그려지면 내부 데이터는 바뀌지 않음
1.  immutable(불변의) - 화면에 보여주는 값이 고정 
2.  UI 전용 컴포넌트에 적합 - 사용자 행동이나 시간 경과에 따라 UI가 변하지 않음 
3.  구조가 단순하고 가벼움 - build()는 입력값이 같으면 항상 같은 UI를 반환. 
## When 
1. 단순 텍스트, 아이콘
2. 버튼 모양만 정의할 때
3. 데이터가 외부에서 내려오기만 하는 경우
## ex
1. 앱 상단 타이틀
2. 고정된 이미지
3. 색상만 있는 컨테이너


# StateFulWidget
----------------
1) 값이 바뀌는가? Y
2) 누가 상태를 관리하는가? 내부 필드에서 직접 상태 관리

: state(상태)를 가지는 위젯 -> 상태가 변경되면 rebuild(화면이 다시 그려짐)
- 사용자 입력, 버튼 클릭, 네트워크 응답, 시간 변화에 따라 UI가 바껴야 할 때 반드시 필요
- StatefulWidget: 틀 / State: 실제 데이터와 로직
- Flutter는 State 객체만 교체하면 화면을 다시 그린다.
